# 🌿 Ecoyaan Checkout

A production-ready, simplified e-commerce checkout flow built with **Next.js 16**, **React 19**, **Tailwind CSS**, and **Zustand** — inspired by Ecoyaan's sustainable shopping experience.

---

## 🚀 Quick Start

```bash
# 1. Install dependencies
npm install

# 2. Run development server
npm run dev
# → Open http://localhost:3000
```

The app starts at `/` and auto-redirects to `/cart`.

---

## 🏗️ Build for Production

```bash
npm run build
npm run start
```

## 🧭 Checkout Flow

```
/cart  →  /address  →  /payment  →  /success
```

| Route | Purpose | Data Source |
|-------|---------|-------------|
| `/cart` | View cart items, totals | **SSR** — Server Component |
| `/address` | Enter shipping details | Zustand (cart) + form input |
| `/payment` | Review order, pay | Zustand (cart + address) |
| `/success` | Order confirmation | Zustand (cart + address + orderId) |

---

## ⚡ Server-Side Rendering (SSR)

The cart page (`/cart/page.tsx`) is a **Next.js App Router Server Component**:

```tsx
// app/cart/page.tsx
export default async function CartPage() {
  // Runs on the server — replaces with real API call in production
  const cartData = await fetchCartData();

  return (
    <CheckoutLayout ...>
      <CartClientShell cartData={cartData} />  {/* passes data to client */}
    </CheckoutLayout>
  );
}
```

**How it works:**
1. `fetchCartData()` in `lib/mockData.ts` simulates an async API call
2. Data is fetched and rendered on the server (zero client JS for initial paint)
3. `CartClientShell` receives the data as a prop and hydrates on the client
4. On mount, it syncs data into the Zustand store for downstream pages

**In production**, replace `fetchCartData()` with:
```ts
const res = await fetch('https://api.ecoyaan.com/cart', { cache: 'no-store' });
const data = await res.json();
```

## 🗄️ State Management (Zustand)

The checkout state lives in `context/CheckoutContext.tsx`:

```ts
interface CheckoutStore {
  cartData: CartData | null;        // Populated by CartClientShell from SSR
  shippingAddress: ShippingAddress | null;  // Set by AddressForm on submit
  orderId: string | null;           // Generated by useProcessPayment()
}
```

**Data flow:**
```
SSR fetch → CartClientShell → setCartData()
AddressForm submit → setShippingAddress()
Pay button → useProcessPayment() → setOrderId() → /success
```

Each downstream page reads from the store and redirects back if required data is missing (route guarding).
